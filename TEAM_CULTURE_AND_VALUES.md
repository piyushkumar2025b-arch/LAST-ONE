# ⬡ ChemoFilter: Team Culture & Core Values (2026-2027)

**The Human Spirit of the "Autonomous Discovery" Era**  
*Building a Multi-Disciplinary Community for ChemoFilter v2.0*

---

## 1. Executive Overview

ChemoFilter was built to be a bridge between high-end scientific modeling and modern human interactivity. This "Culture Doc" was chosen for its **Transparency**, **Privacy**, and **Scientific Rigor**.

---

## 2. Core Scientific Values (The Logic)

1.  **"Transparency over Black-Boxes":** We believe every calculation (Tier 1-15) must be explained in plain English. No lead is accepted "just because the AI said so."
2.  **"Privacy-First Discovery":** We value the Intellectual Property of every researcher. Structural data should never leave the researcher's machine unless explicitly allowed.
3.  **"Democratizing High-End Tools":** We aim to bring Schrödinger-grade capability to every lab, regardless of budget or coding ability.

---

## 3. Core Software Values (The Backbone)

1.  **"Efficiency as a Feature":** We believe high-end tools should run on standard hardware. $O(1)$ memory scaling is a first-tier requirement for every new module.
2.  **"Modernity as a Requirement":** We believe scientific tools shouldn't look like they were built in the 1990s. The **Crystalline Obsidian** UI is as important as the RDKit kernel.
3.  **"Open-Source Integrity":** While we build for institutions, we maintain a pure open-source core to ensure trust and communal contribution.

---

| Value | Component | Human Focus |

## 5. Community & Collaboration post-MDP 2026

The `culture_registry.py` module tracks all developer contributions for a potential global launch. To join the team:
1.  Navigate to `CONTRIBUTING.md`.
2.  Propose a new way to democratize discovery (e.g. **Mobile Research**).
3.  Submit a Pull Request (PR) after passing the **"Vanguard Sanity Check"** in `app.py`.
